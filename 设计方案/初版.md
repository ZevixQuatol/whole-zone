# 寰域（Whole-Zone）分享社区设计方案

## 一、项目概述

### 1.1 产品定位

寰域是一个新时代的信息分享社区平台，核心理念是**"标签连接万物，探索无限可能"**。平台致力于打破传统社区的信息孤岛模式，通过标签系统将所有用户、内容和话题有机连接，形成一个开放、互联的知识共享生态。

### 1.2 核心价值观

- **开放共享**：所有用户在同一网络、同一数据库中畅所欲言
- **知识传递**：鼓励知识分享，促进共同进步
- **探索发现**：通过标签关联帮助用户发现感兴趣的内容
- **圈层交流**：支持用户创建专属领域，结识志同道合的朋友

## 二、核心概念设计

### 2.1 标签系统（Tag System）

标签是整个社区的基础原子单元，用于定义和关联一切事物。

#### 2.1.1 标签特性

| 特性 | 说明 |
|------|------|
| 多维性 | 单一事物可拥有多个标签，如同多维向量 |
| 关联性 | 标签之间可建立关联关系，形成知识图谱 |
| 成长性 | 标签随着用户使用而发展、演化 |
| 层级性 | 支持父标签、子标签的层级结构 |

#### 2.1.2 标签类型

```
标签类型体系：
├── 基础标签
│   ├── 人物标签（职业、身份）
│   ├── 事物标签（物品、概念）
│   ├── 地点标签（国家、城市、区域）
│   ├── 时间标签（历史事件、节日）
│   └── 技能标签（编程语言、工具）
├── 复合标签
│   ├── 领域标签（技术、艺术、商业）
│   ├── 兴趣标签（爱好、运动、娱乐）
│   └── 事件标签（活动、新闻、话题）
└── 自定义标签
    └── 用户创建的个性化标签
```

#### 2.1.3 标签关联示例

```
示例：苹果的标签关联链

苹果
├── 水果（父类别）
│   ├── 食物 → 饮食 → 生活
│   └── 农产品 → 农业 → 经济
├── 苹果公司（品牌）
│   ├── 科技公司 → 互联网 → 数字经济
│   └── iPhone → 智能手机 → 电子产品
├── 红富士（品种）
│   ├── 洛川苹果（产地）
│   │   ├── 洛川（陕西）
│   │   └── 延安（陕西）
│   └── 静宁苹果（产地）
│       ├── 静宁（甘肃）
│       └── 兰州（甘肃）→ 朋友所在地
```

### 2.2 探索引擎（Explore Engine）

模拟推荐算法逻辑，帮助用户在标签的关联网络中探索未知领域。

#### 2.2.1 探索机制

- **标签漫游**：从当前标签出发，探索关联标签
- **兴趣扩散**：基于用户兴趣标签，推荐相关联的内容
- **热度追踪**：追踪标签热度变化，发现新兴话题
- **路径记忆**：记录用户探索路径，提供回溯功能

#### 2.2.2 探索算法

```
探索推荐算法伪代码：

function explore(currentTag, userProfile):
    relatedTags = getRelatedTags(currentTag)
    scoredTags = []
    
    for tag in relatedTags:
        score = calculateRelevanceScore(tag, userProfile)
        score += tag.popularity * 0.3
        score += tag.freshness * 0.2
        score += tag.userEngagement * 0.2
        scoredTags.append(score)
    
    return sortByScore(scoredTags).topN(10)
```

### 2.3 寰（World）- 全局社区

所有用户共享的公共空间，类似于社区的"广场"。

#### 2.3.1 寰的功能

- **全局动态**：所有用户的公开分享、讨论
- **热门话题**：基于标签热度的话题聚合
- **知识广场**：按领域分类的知识分享
- **全站搜索**：跨标签、跨域的内容检索

### 2.4 域（Domain）- 私人圈子

用户自定义的专属领域，基于兴趣或目的形成的交流圈。

#### 2.4.1 域的特性

| 特性 | 说明 |
|------|------|
| 标签组成 | 由一个或多个标签定义域的主题 |
| 成员管理 | 域主可以邀请、审核成员 |
| 隐私控制 | 公开域/私密域可选 |
| 域内规则 | 域主可制定域内交流规则 |

#### 2.4.2 域的创建流程

```
创建域流程：
1. 选择基础标签（至少1个）
2. 填写域信息（名称、描述、头像）
3. 设置隐私级别（公开/私密/邀请制）
4. 制定域规（可选）
5. 邀请成员（可选）
```

### 4.2 内容系统

| 类型 | 说明 | 支持格式 |
|------|------|----------|

## 五、数据模型设计

### 5.1 核心实体

#### 5.1.1 用户（User）

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE,
    phone VARCHAR(20),
    password_hash VARCHAR(255) NOT NULL,
    avatar_url VARCHAR(500),
    bio TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT 'active'
);
```

#### 5.1.2 标签（Tag）

```sql
CREATE TABLE tags (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) UNIQUE NOT NULL,
    description TEXT,
    parent_id UUID REFERENCES tags(id),
    level INT DEFAULT 0,
    usage_count INT DEFAULT 0,
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT 'active'
);

CREATE TABLE tag_relations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    source_tag_id UUID REFERENCES tags(id),
    target_tag_id UUID REFERENCES tags(id),
    relation_type VARCHAR(50),
    weight FLOAT DEFAULT 1.0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 5.1.3 内容（Content）

```sql
CREATE TABLE contents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    author_id UUID REFERENCES users(id),
    title VARCHAR(500),
    body TEXT,
    content_type VARCHAR(20) NOT NULL,
    visibility VARCHAR(20) DEFAULT 'public',
    domain_id UUID REFERENCES domains(id),
    view_count INT DEFAULT 0,
    like_count INT DEFAULT 0,
    comment_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT 'published'
);

CREATE TABLE content_tags (
    content_id UUID REFERENCES contents(id),
    tag_id UUID REFERENCES tags(id),
    PRIMARY KEY (content_id, tag_id)
);
```

#### 5.1.4 域（Domain）

```sql
CREATE TABLE domains (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    description TEXT,
    owner_id UUID REFERENCES users(id),
    privacy VARCHAR(20) DEFAULT 'public',
    avatar_url VARCHAR(500),
    member_count INT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT 'active'
);

CREATE TABLE domain_tags (
    domain_id UUID REFERENCES domains(id),
    tag_id UUID REFERENCES tags(id),
    PRIMARY KEY (domain_id, tag_id)
);

CREATE TABLE domain_members (
    domain_id UUID REFERENCES domains(id),
    user_id UUID REFERENCES users(id),
    role VARCHAR(20) DEFAULT 'member',
    joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (domain_id, user_id)
);
```

## 九、总结

寰域（Whole-Zone）作为一个新时代的分享社区，通过**标签系统**将知识、内容和用户有机连接，让信息探索变得更加自然和高效。**寰**代表了开放的全局社区，**域**代表了私密的交流圈子，两者结合形成了一个既有广度又有深度的知识共享生态。

本设计方案从产品理念、核心功能、技术架构、数据模型、界面设计等多个维度进行了全面规划，为后续开发提供了清晰的指导方向。
