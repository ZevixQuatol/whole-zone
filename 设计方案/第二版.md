# 寰域（Whole-Zone）分享社区设计方案（第二版）

## 一、项目概述

### 1.1 产品定位

寰域是一个新时代的信息分享社区平台，核心理念是**"标签连接万物，探索无限可能"**。平台通过智能标签系统、用户粘性机制和开放的域生态，构建一个自我演化、精准推荐的知识共享社区。

### 1.2 核心价值观

- **开放共享**：所有用户在同一网络、同一数据库中畅所欲言
- **知识传递**：鼓励知识分享，促进共同进步
- **智能演化**：标签随用户行为自动生长、淘汰
- **精准匹配**：基于粘性画像实现按需推送

---

## 二、核心概念设计

### 2.1 标签系统（Tag System）

标签是整个社区的基础原子单元，承载着内容分类、用户画像、推荐算法的核心职能。

#### 2.1.1 标签特性

| 特性 | 说明 |
|------|------|
| 多维性 | 单一事物可拥有多个标签，如同多维向量 |
| 关联性 | 标签之间可建立关联关系，形成知识图谱 |
| 成长性 | 标签随着用户使用而发展、演化 |
| 粘性值 | 每个用户对每个标签有独立的粘性数值 |
| 生命周期 | 标签有热度周期，会被系统回收 |

#### 2.1.2 标签来源

```
标签创建流程：

1. 智能生成（系统侧）
   └── 用户发布内容时，AI分析标题和内容
       → 向量化处理 → 提取关键词 → 生成候选标签

2. 用户自定义（用户侧）
   └── 用户可自行创建新标签
       → 进入待审核队列 → 审核通过后生效

3. 域标签池
   └── 标签发布后进入当前域的标签池
       → 域内用户共同维护
```

#### 2.1.3 标签评级体系

用户可对内容中的标签进行8级评级：

| 评级 | 粘性变化 | 含义 |
|------|----------|------|
| 屏蔽 | -10 | 永久屏蔽该标签相关内容 |
| 厌恶 | -5 | 不喜欢该标签 |
| 不喜 | -2 | 有点不喜欢 |
| 无感 | 0 | 无所谓 |
| 见识 | +1 | 了解一下 |
| 有趣 | +3 | 觉得有趣 |
| 认可 | +5 | 认可该标签内容 |
| 钟爱 | +10 | 非常喜欢，持续关注 |

#### 2.1.4 标签热度管理

```
热度计算规则：

1. 统计周期：7天、10天、30天
2. 热度因子：
   - 内容发布量 × 1.0
   - 互动量（点赞+评论+收藏）× 2.0
   - 新用户关注量 × 3.0

3. 回收机制：
   - 连续30天热度低于阈值 → 进入回收观察期
   - 观察期7天内热度回升 → 恢复显示
   - 观察期仍未改善 → 系统回收，不再显示
```

#### 2.1.5 标签关联管理

- 用户可对标签之间的关联度进行评级
- 系统定期统计关联度评分
- 低关联度标签关系自动解除
- 高关联度标签关系强化推荐

---

### 2.2 用户粘性系统（Tag Adhesion）

用户与标签之间通过"粘性"建立连接，粘性值决定用户的内容推荐和域成员身份。

#### 2.2.1 粘性计算

```
粘性变化规则：

1. 阅读内容
   └── 阅读时关联标签 +1

2. 标签评级
   └── 正面评级（见识~钟爱）→ 粘性增加
   └── 负面评级（屏蔽~不喜）→ 粘性减少

3. 发布内容
   └── 发布带某标签内容 → 该标签粘性 +2

4. 持续关注
   └── 连续7天阅读同类内容 → 粘性加速增长
```

#### 2.2.2 用户画像形成

```
用户画像数据结构：

UserProfile {
    userId: string
    username: string
    avatar: string
    bio: string
    
    // 标签粘性图谱
    tagAdhesions: {
        "Python": 85,      // 粘性值 0-100
        "旅行": 62,
        "摄影": 45,
        "美食": 30,
        ...
    }
    
    // 画像标签（粘性>50的标签）
    profileTags: ["Python", "旅行", ...]
    
    domains: [domainId, ...]    // 所属域
    following: [userId, ...]    // 关注列表
    createdAt: timestamp
}
```

### 2.3 探索引擎（Explore Engine）

帮助用户在标签的关联网络中探索未知领域，解决"不知道想看什么"的困惑。

#### 2.3.1 探索机制

- **标签漫游**：从当前标签出发，探索关联标签
- **兴趣扩散**：基于用户粘性画像，推荐相关联的内容
- **热度追踪**：追踪标签热度变化，发现新兴话题
- **路径记忆**：记录用户探索路径，提供回溯功能

#### 2.3.2 探索触发场景

| 场景 | 触发条件 | 推荐策略 |
|------|----------|----------|
| 主动探索 | 用户明确搜索标签 | 展示关联标签树 |
| 被动探索 | 用户无明确方向 | 基于粘性推荐 |
| 冷启动 | 新用户无粘性数据 | 热门标签推荐 |
| 域内探索 | 在某个域内浏览 | 域标签关联推荐 |

### 2.4 寰（World）- 全局社区

所有用户共享的公共空间，类似于社区的"广场"。

#### 2.4.1 寰的功能

- **全局动态**：所有用户的公开分享、讨论
- **热门话题**：基于标签热度的话题聚合
- **知识广场**：按领域分类的知识分享
- **全站搜索**：跨标签、跨域的内容检索

---

### 2.5 域（Domain）- 私人圈子

用户基于共同兴趣标签形成的交流圈，是社区的核心组织单元。

#### 2.5.1 域的创建规则

```
创建域流程：

1. 选择基础标签（至少1个，最多5个）
   └── 这些标签将定义域的主题

2. 填写域信息
   └── 名称、描述、头像

3. 域只能是开放状态
   └── 所有用户可见，所有用户可进入

4. 创建完成
   └── 进入域标签池管理
```

#### 2.5.2 域成员机制

```
成员等级体系：

1. 核心成员
   └── 判断条件：
       - 标签匹配100%（拥有域所有标签）
       - 粘性达到域用户前80%水平
   └── 权益：
       - 域内最高发言权重
       - 可参与域管理投票

2. 普通成员
   └── 满足基本标签匹配
   └── 权益：
       - 正常浏览、发布、互动

3. 访客
   └── 尚未匹配域标签的用户
   └── 权益：
       - 可浏览公开内容
       - 申请加入域
```

#### 2.5.3 域的流动机制

```
跨域流动：

1. 自动匹配
   └── 用户接触新标签时
   └── 系统匹配用户与各域的标签重合度
   └── 100%重合时自动成为核心成员

2. 邀请机制
   └── 核心成员可邀请其他用户
   └── 被邀请用户直接进入域
   └── 但粘性需自行培养

3. 退出机制
   └── 用户可主动退出域
   └── 长期不活跃自动退出
```

#### 2.5.4 域的生命周期

```
域的存续规则：

1. 活跃判断
   └── 统计周期：100天
   └── 判断因子：
       - 活跃用户数量
       - 用户平均粘性程度

2. 回收机制
   └── 连续100天活跃度低于阈值
   └── 系统自动回收，域关闭

3. 域管理员权限
   └── 可增删域内的标签
   └── 不可直接删除用户
   └── 异常用户提交官方处理
```

## 三、用户交互设计

### 3.1 内容阅读交互

```
用户阅读文章时：

1. 标签关联
   └── 系统记录用户阅读了哪些标签
   └── 对应标签粘性 +1

2. 标签评级
   └── 用户可对每个标签进行8级评级
   └── 评级影响粘性值

3. 作者关注
   └── 用户可关注作者
   └── 关注后收到全量推送
   └── 或选择只接收特定标签内容推送
```

### 3.2 内容发布交互

```
发布流程：

1. 智能标签生成
   └── AI分析标题和内容
   └── 自动提取关键词生成候选标签

2. 标签调整
   └── 用户可增删改标签
   └── 最多添加10个标签

3. 选择发布范围
   └── 公开（发布到寰）
   └── 域（发布到指定域）

4. 发布
   └── 标签进入当前域的标签池
   └── 内容进入审核队列
```

## 六、核心算法设计

### 6.1 标签智能生成算法

```python
def generate_tags(title: str, content: str) -> List[Tag]:
    """
    智能标签生成
    """
    # 1. 文本预处理
    text = preprocess(title + " " + content)
    
    # 2. 关键词提取（TF-IDF + 词性分析）
    keywords = extract_keywords(text)
    
    # 3. 向量化
    vectors = embed(keywords)
    
    # 4. 与现有标签匹配
    similar_tags = find_similar_tags(vectors)
    
    # 5. 生成候选标签
    candidates = keywords + similar_tags
    
    # 6. 去重、排序返回
    return deduplicate_and_rank(candidates)
```

### 6.2 粘性计算算法

```python
def calculate_adhesion(user_id: str, tag_id: str) -> int:
    """
    计算用户对标签的粘性值
    """
    adhesion = 0
    
    # 基础阅读粘性
    read_count = get_read_count(user_id, tag_id)
    adhesion += read_count * 1
    
    # 评级粘性
    ratings = get_ratings(user_id, tag_id)
    for rating in ratings:
        adhesion += RATING_WEIGHTS[rating]  # 评级权重表
    
    # 发布粘性
    publish_count = get_publish_count(user_id, tag_id)
    adhesion += publish_count * 2
    
    # 持续关注加成
    if is_continuous_reading(user_id, tag_id, days=7):
        adhesion *= 1.5
    
    return clamp(adhesion, 0, 100)  # 限制在0-100
```

### 6.3 核心成员判定算法

```python
def is_core_member(user_id: str, domain_id: str) -> bool:
    """
    判断用户是否为域的核心成员
    """
    domain = get_domain(domain_id)
    required_tags = domain.required_tags
    
    # 1. 标签匹配100%
    user_tags = get_user_tags(user_id)
    if not all(tag in user_tags for tag in required_tags):
        return False
    
    # 2. 粘性达到前80%
    member_adhesions = get_domain_member_adhesions(domain_id)
    user_adhesion = sum(get_adhesion(user_id, tag) for tag in required_tags)
    
    threshold = calculate_percentile(member_adhesions, 80)
    
    return user_adhesion >= threshold
```

### 6.4 推荐算法

```python
def recommend_content(user_id: str, limit: int = 20) -> List[Content]:
    """
    个性化内容推荐
    """
    user_profile = get_user_profile(user_id)
    
    # 获取用户高粘性标签
    high_adhesion_tags = get_high_adhesion_tags(user_id, threshold=50)
    
    # 候选内容评分
    candidates = get_recent_contents()
    scored_contents = []
    
    for content in candidates:
        score = 0
        
        # 标签匹配度
        content_tags = get_content_tags(content.id)
        for tag in content_tags:
            adhesion = get_adhesion(user_id, tag.id)
            score += adhesion * 0.5
        
        # 热度因子
        score += content.hot_score * 0.2
        
        # 作者关注因子
        if is_following(user_id, content.author_id):
            score += 20
        
        # 去重（避免重复推荐）
        if has_recommended_before(user_id, content.id):
            score *= 0.5
        
        scored_contents.append((content, score))
    
    # 排序返回
    scored_contents.sort(key=lambda x: x[1], reverse=True)
    return [c[0] for c in scored_contents[:limit]]
```

## 十、总结

寰域（Whole-Zone）作为一个新时代的分享社区，通过**智能标签系统**和**用户粘性机制**，实现了内容的自我演化、精准推荐和社区自治。

**核心创新点：**

1. **标签粘性系统** - 用户行为自动形成个性化标签粘性画像
2. **8级评级机制** - 让用户精准表达内容偏好
3. **核心成员制度** - 标签匹配+粘性判定，形成自治生态
4. **域生命周期** - 100天活跃度判定，自动回收低质量域
5. **跨域流动** - 标签匹配触发自动加入，实现用户自然流转
