# 寰域 whola-zone

新时代的分享社区，共同参与，共同管理
> 不要吝啬知识的分享，只要你学习、进步速度够快，就没有人能够追得上你。


**标签**
-  一份职业，一个领域，一种编程语言，一种技术框架，万事万物皆有标签来定义，类似维度、向量的定义  。

- 一切事物都可以用多个标签来“定义”，就像说起水果会想到苹果，想到红富士（品种），想到洛川、静宁（产地），想到陕西、甘肃（省），思维就这样发散出去；想到甘肃，想到兰州，碰巧我有一个朋友在兰州，思维回到我自己身上。  

**探索**
- 参考推荐算法，从一个标签出发，关联到另一个标签，如今大数据推荐，越来越精准，越来越深入你心。面对扑面而来的信息，你是否也会迷茫，阅读不过来，甚至觉得这些被推荐来的信息，占据了太多你的关注。
- 很多时候，想要了解怎么样的信息，或许你心里并没有一个明确的方向；在标签和标签的关联下，不断探索，知道某个事情引起你持续的关注；它可以使爱好，可以是提升自我的知识、技能，也可以是寻找志同道合的朋友之法门。

# 寰
- 所有用户都在同一个网络，同一数据库，同一服务器，开怀畅聊，发表自己的看法，记录自己的生活，分享学习的知识。
- 标签的产生、发展、留存，由这个世界所有的用户参与。
# 域
- 你有自己的圈子，或者你想在某些固定的话题内，认识朋友，交流心得，选择你认可的标签，组成专属于你的**域**
# 构建
- 起初，网站没有标签，用户发布内容时，根据文章标题和内容智能分析、拆解、向量化形成标签，用户也可自定义增删改查标签，发布后标签进入当前域的标签池。
# 发展
- 在寰或者某一个单一的域内，用户阅读他人发布的文章内容，此时会发生什么：
```
1. 当前用户关联文章的标签，标签粘性+1。
2. 用户给文章标签评级，或是视频对标签的喜爱程度，赞同喜爱的标签，粘性+1；不感兴趣标签粘性-1；评级由低到高可分为：屏蔽、厌恶、不喜、无感、见识、有趣、认可、钟爱。
3. 用户想要持续关注作者的动态，可以专注文章作者，作者发布新内容将会推送至用户；用户可能只想关注作者某一些标签的持续动态，在订阅时，选择作者的标签则只推送对应标签的内容。
```
- 标签会越来越多，越来越乱，此时用户可能也不知道应该如何探索，才能寻找到自己想要的内容怎么办：
```
1. 服务器定期对标签评级，分别统计7天、10天、30天每个标签的热度，对于热度低于一定标准的标签，系统进行回收，不再显示在标签池中。
2. 同样的，对于标签与标签之间的关联度，用户也可以评级，同样根据一定周期进行评级、回收处理。
```
# 延申
- 用户阅览的内容越多，涉及到的标签就越多；重复类型的内容越多，某些标签的粘性就会越高，这样用户的标签自然而然就行成了。
- 用户与用户之间，就可以通过这些标签，寻找到自己志同道合的朋友；
- 根据用户标签的粘性，系统推送的内容也就越来越精准，按需推送。
# 流动
- 每当用户接触到新的标签时，会对这个标签所关联的域进行匹配，当用户与某个域中的标签完全重合时，用户则自动成为这个域的核心成员，这样用户就从一个域冲入到另一个域。
- 已经属于某个域的核心成员可以邀请其他用户，被邀请用户直接进入域，但是与当前域的标签粘性并不高，所以判断用户是否为域的核心成员的方式为：标签匹配100%并且粘性达到所有域用户粘性程度的80%。
- 用户可以创建专属自己的域
1. 域只能为开放状态，所有用户可见，所有用户可进入，选择专属的标签。
2. 域的存在与否，同样是由系统判断：根据活跃用户数量，以及域中标签与用户粘性程度，若持续100天均较低，则回收域。
3. 域的管理员可以对域中的标签进行增删，管理员不可直接删除域的用户，若存在异常用户（广告，非法传播等），提交至官方系统统一处理。
4. 官方认定的从事广告，刻意破坏等无道德行为进行最高无限期封禁；对于存在违法行为的进行永久封禁，并将注册信息计入黑名单池，不可再用相关信息再次进行注册。
